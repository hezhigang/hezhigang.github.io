Git Pull Vs Git Fetch

Fetch

git fetch really only downloads new data from a remote repository - but it doesnРђЎt integrate any of this new data into your working files. Fetch is great for getting a fresh view on all the things that happened in a remote repository.
Due to itРђЎs РђюharmlessРђЮ nature, you can rest assured: fetch will never manipulate, destroy, or screw up anything.

Pull

git pull, in contrast, is used with a different goal in mind: to update your current HEAD branch with the latest changes from the remote server. This means that pull not only downloads new data; it also directly integrates it into your current working copy files. This has a couple of consequences:

  • Since Рђюgit pullРђЮ tries to merge remote changes with your local ones, a so-called Рђюmerge conflictРђЮ can occur. Check out our in-depth tutorial on How to deal with merge conflicts for more information.
  • Like for many other actions, itРђЎs highly recommended to start a Рђюgit pullРђЮ only with a clean working copy. This means that you should not have any uncommitted local changes before you pull.
